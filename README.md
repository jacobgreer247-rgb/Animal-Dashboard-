1. How do you write programs that are maintainable, readable, and adaptable? Especially consider your work on the CRUD Python module from Project One, which you used to connect the dashboard widgets to the database in Project Two. What were the advantages of working in this way? How else could you use this CRUD Python module in the future?

I write programs that are maintainable, readable, and adaptable by breaking larger tasks into smaller pieces and keeping my code organized. In Project One, creating the CRUD Python module allowed me to separate the database operations from the rest of the program, which made it easier to test, update, and reuse the code later in Project Two.

One of the biggest advantages of working this way was that I did not have to rewrite my database functions when creating the dashboard. Instead, I was able to import the CRUD module and focus on building the dashboard features and visualizations. In the future, I could use this same CRUD module structure in other projects that require a database connection because it provides a reusable and organized way to manage database operations.
